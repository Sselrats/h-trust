# H-TRUST 구현 프롬프트

**발표용 단일 페이지 인터랙티브 데모 웹사이트**를 만들어줘.
목표는 **H-TRUST의 금융 AI 의사결정 파이프라인**을 시각적으로 설명하는 것이다.

이 데모의 목적은 **모델 성능을 보여주는 것**이 아니라, 아래 구조를 직관적으로 보여주는 것이다.

```text
Domain AI → TRUST Layer → 고객/직원 전달
```

핵심 메시지는 다음과 같다.

- 금융 AI는 점점 더 많은 결정을 내린다.
- 금융사는 false positive를 줄이기 위해 보수적으로 모델을 설계할 수밖에 없다.
- 그 결과 고객 입장에서는 이해하기 어려운 **“No”**가 늘어난다.
- **H-TRUST는 AI의 판단을 설명 가능하고 관리 가능한 결정으로 바꾸는 금융 AI Trust Infrastructure**다.

기술 스택은 다음을 사용해줘.

- **Next.js**
- **React**
- **Tailwind CSS**
- 필요하면 **Framer Motion**

전체 페이지는 **한 페이지(single-page)**로 만들고, 발표장에서 보기 쉽게 **깔끔한 금융 대시보드 스타일**로 만들어줘.

---

## 전체 페이지 구조

페이지는 아래 4개 섹션으로 구성해줘.

1. Header
2. Scenario Selector
3. Main Pipeline Area
4. Bottom Insight Area

가능하면 **노트북 화면 기준 한 화면에 대부분 들어오도록** 구성해줘.

---

## 1. Header

상단 중앙에 제목과 설명을 배치해줘.

### 제목

**H-TRUST Decision Flow Demo**

### 부제

**금융 AI는 어떻게 “No”를 전달하는가**

### 설명 문구

**Domain AI가 판단하고, TRUST Layer가 그 판단을 설명 가능하고 관리 가능한 결과로 바꿉니다.**

스타일은 너무 화려하지 않게, 발표용으로 또렷하게 보여야 한다.

---

## 2. Scenario Selector

헤더 아래에는 **3개의 시나리오 선택 카드**를 배치해줘.

### 시나리오 목록

1. 보험금 지급
2. 대출 심사
3. 투자 제한

각 카드에는 다음이 들어가야 한다.

- 시나리오 제목
- 한 줄 설명

예시:

- 보험금 지급
  `보험금 청구가 보류 또는 거절될 수 있는 상황`
- 대출 심사
  `대출 신청이 심사 모델에 의해 부결될 수 있는 상황`
- 투자 제한
  `고객 투자 성향상 고위험 상품 가입이 제한되는 상황`

### 동작

- 카드 클릭 시 선택 상태가 강조되어야 한다.
- 선택된 시나리오 데이터가 로드되어야 한다.
- 아래에 **“파이프라인 실행”** 버튼을 두고, 이 버튼을 누르면 메인 파이프라인이 순차적으로 나타나야 한다.

---

## 3. Main Pipeline Area

이 영역이 핵심이다.

가로 방향 4단계 파이프라인으로 보여줘.

```text
[입력] → [Domain AI] → [TRUST Layer] → [최종 전달]
```

각 단계는 카드 형태로 구성하고,
**파이프라인 실행 버튼 클릭 시 왼쪽에서 오른쪽으로 순차적으로 애니메이션 등장**하게 해줘.

각 카드의 높이와 스타일은 최대한 통일해줘.

---

### Step 1. 입력 카드

제목: **입력 상황**

선택한 시나리오의 실제 입력 상황을 보여줘.

예:

- 고객 요청 요약
- 주요 입력 정보 3~4개

표현 방식은 라벨-값 형태가 좋다.

---

### Step 2. Domain AI 카드

제목: **Domain AI 판단**

여기에는 고객에게 직접 보여주지 않는 **내부 판단 결과**를 보여줘.

표시 항목:

- 모델명
- 내부 사유 코드(reason codes)
- 리스크 점수
- 판단 결과

예:

- 모델명: Claim Risk Model
- 내부 사유 코드:
  - OUT_OF_COVERAGE
  - FRAUD_RISK_PATTERN

- 리스크 점수: 82/100
- 판단 결과: HOLD

이 카드는 **내부 시스템 화면처럼 조금 더 기술적인 느낌**이 나야 한다.

---

### Step 3. TRUST Layer 카드

제목: **TRUST Layer**

이 카드는 H-TRUST의 핵심 기능을 보여준다.
카드 안에 아래 4개의 작은 섹션을 넣어줘.

#### 1) 사유 정리

내부 사유 코드를 사람이 이해할 수 있는 표현으로 정리

#### 2) 설명 생성

고객에게 보여줄 설명 초안 생성

#### 3) HITL 라우팅

사람 검토 필요 여부 표시

#### 4) 후속 액션

고객이 취할 수 있는 다음 행동 제시

예:

- 추가 서류 제출
- 재검토 요청
- 상담 연결
- 대안 상품 확인

---

### Step 4. 최종 전달 카드

제목: **고객 전달 메시지**

여기서는 실제 고객이 받는 최종 메시지를 보여줘.

구성:

- 상태 배지 (예: 지급 보류, 심사 부결, 투자 제한)
- 자연어 메시지
- 아래 액션 버튼 2~3개

예시 버튼:

- 재검토 요청
- 추가 서류 제출
- 상담사 연결
- 대안 상품 보기

이 카드는 **고객 친화적이고 가장 읽기 쉬운 카드**여야 한다.

---

## 4. Bottom Insight Area

파이프라인 아래에 보조 설명 영역을 넣어줘.

2개의 카드 또는 탭으로 구성해줘.

### A. 왜 중요한가

다음 메시지를 짧게 보여줘.

- 금융 AI는 false positive를 줄이기 위해 보수적으로 설계된다.
- 그 결과 고객 입장에서는 이해하기 어려운 “No”가 늘어난다.
- H-TRUST는 이 결정을 설명 가능하고 관리 가능한 형태로 바꾼다.

### B. Feedback Loop

간단한 순환 구조를 보여줘.

```text
고객 요청
→ Domain AI 판단
→ TRUST 설명/전달
→ 재검토·민원 발생
→ 데이터 축적
→ Domain AI 및 정책 개선
```

짧은 설명:
**H-TRUST는 어려운 의사결정 사례를 구조화된 학습 데이터로 전환합니다.**

---

## 인터랙션 요구사항

- 사용자가 시나리오를 하나 선택한다.
- “파이프라인 실행” 버튼 클릭 시:
  - 입력 카드 표시
  - 0.7초 후 Domain AI 카드 표시
  - 0.7초 후 TRUST Layer 카드 표시
  - 0.7초 후 최종 전달 카드 표시

- 애니메이션은 과하지 않고 깔끔해야 한다.
- 전체 페이지는 발표용이므로 조작이 단순해야 한다.

---

## 디자인 가이드

- 배경: 밝은 톤
- 포인트 컬러: 네이비 / 블루
- 카드: 둥근 모서리, 얕은 그림자
- 위험 강조: 빨강보다 앰버/오렌지 계열
- 폰트 크기: 발표장에서 잘 보여야 하므로 일반 서비스보다 크게
- 전체적으로 **금융사 내부 데모 대시보드** 느낌

피해야 할 것:

- 채팅 UI 중심 구조
- 너무 많은 색상
- 해커톤 느낌의 로그 화면
- 과한 글래스모피즘

---

## 핵심 목적

사용자가 이 데모를 보고 바로 이해해야 하는 핵심은 이것이다.

```text
Domain AI는 판단을 만든다.
TRUST Layer는 그 판단을 설명 가능하고 관리 가능한 결과로 바꾼다.
```

즉, 이 데모는 **금융 AI의 판단을 어떻게 “납득 가능한 전달”로 바꾸는지**를 보여줘야 한다.
